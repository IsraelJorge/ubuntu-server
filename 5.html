O ProFTPD (Professional FTP Daemon) é um servidor FTP altamente configurável e eficiente, amplamente utilizado em sistemas Unix e similares, incluindo Linux. Este servidor foi desenvolvido com foco em segurança, desempenho e flexibilidade, tornando-se uma escolha popular para administradores de sistemas e provedores de serviços de internet (ISPs) que necessitam de um servidor FTP robusto e confiável.

5.1 Instalação

Primeiramente, você deve acessar seu servidor com privilégios de superusuário. Depois, utilize o comando apt para instalar o pacote ProFTPD e sua documentação. 

    sudo su
    apt install proftpd proftpd-doc

5.2 Configurando o ProFTPD

1 - Navegue até o diretório de configuração do ProFTPD:

  cd /etc/proftpd

2 - Faça um cópia de segurança do arquivo de configuração padrão:

  cp proftpd.conf proftpd.conf.backup

3 - Edite o arquivo de configuração do ProFTPD:

  nano proftpd.conf

Dentro do arquivo, altere as seguintes configurações:

Configurações feitas:

  • UseIPv6 off: Desligue o suporte a IPv6 para simplificar a configuração do sistema.
  • ServerName "israel-server": Especifique o nome do servidor FTP, substituindo "israel-server" pelo nome desejado.
  • DefaultRoot ~: Defina o diretório raiz padrão para os usuários FTP, restringindo o acesso apenas a esse diretório.
  • RequireValidShell off: Permita que usuários com shells inválidos se conectem, útil para usuários FTP que não necessitam de acesso ao shell.
  • MaxInstances 2: Limite o número de conexões simultâneas a 2 (ajuste este valor conforme necessário).

5.3 Adicionando um usuário

Vamos adicionar um novo usuário FTP utilizando o comando:

   sudo adduser user1

5.4 Adicionando uma pasta para o usuário

Vamos acessar o diretório do novo usuário e configurar uma pasta específica para FTP. 

    cd /home/user1
    mkdir ftp

Vamos modificar o arquivo /etc/passwd para definir o diretório FTP e impedir o login do usuário. 

  nano /etc/passwd

Encontre a linha correspondente ao usuário user1 e modifique-a para: 

   user1:x:1001:1001:Usuario FTP 01,,,:/home/user1/ftp:/bin/nologin

Ajuste as permissões da pasta ftp:

    chown user1:user1 ftp
    chmod 770 ftp

5.5 Reiniciando o ProFTPD

Depois de realizar todas as configurações, é necessário reiniciar o ProFTPD para que as alterações entrem em vigor. 

systemctl restart proftpd

Verifique o status do ProFTPD para garantir que esteja funcionando corretamente: 

systemctl status proftpd

5.6 Testando a Configuração

Vamos criar alguns arquivos de teste no diretório FTP:

    touch ftp/naruto.mov
    touch ftp/lobo-pidao.jpg

Ao seguir estes passos, você terá instalado e configurado o servidor FTP ProFTPD no seu Ubuntu Server. Essas configurações iniciais asseguram que o servidor esteja preparado para realizar transferências de arquivos de maneira segura e eficiente.
